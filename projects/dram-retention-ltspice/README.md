# DRAM 保持时间 LTspice 项目

## 目标

用简化的电容和漏电阻模型理解 DRAM 存储节点为什么会随时间放电，以及电容量和漏电如何
影响保持时间。本项目是原理模型，不代表真实 DRAM 单元结构或生产参数。

## 文件

- `dram_retention.cir`：可由 LTspice 打开的 SPICE 网表。

## 操作步骤

1. 安装 LTspice。
2. 打开 `dram_retention.cir`。
3. 运行 transient analysis。
4. 绘制 `V(storage_good)` 和 `V(storage_leaky)`。
5. 测量两个节点下降到 0.5 V 所需时间。
6. 分别修改 `Ccell` 和两个漏电阻，记录保持时间变化。

## 必做实验

| 实验 | 修改 | 预期观察 |
| --- | --- | --- |
| 基准 | 不修改 | 漏电更大的节点下降更快 |
| 电容加倍 | `Ccell=60f` | 保持时间增加 |
| 漏电减小 | 电阻提高 10 倍 | 保持时间明显增加 |
| 温度讨论 | 不强行仿真 | 查资料解释高温为何通常增加漏电 |

## 项目报告结构

1. DRAM 1T1C 单元的作用。
2. 简化模型与假设。
3. 四组波形和测量结果。
4. RC 时间常数与保持时间的关系。
5. 模型局限：未包含访问晶体管、感放、位线和真实漏电机制。
6. 对刷新周期、功耗和可靠性的理解。
