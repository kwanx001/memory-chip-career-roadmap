# 8D 失效分析作品

这里提供两个模拟案例。目标是训练结构化调查方法，不是背诵一个未经验证的“标准答案”。

## 案例一：NAND 编程失败率升高

模拟现象：

- 某批 NAND 在高温测试后编程失败率升高。
- 异常集中在两个 wafer 和特定区域。
- 当前没有足够证据判断是器件、工艺、测试还是量测问题。

使用 `nand-program-failure-template.md` 完成调查。

建议验证方向：

- retest 与换机台验证。
- wafer map、lot、设备、时间和配方分层。
- 编程/擦除循环与温度对失效率的影响。
- I-V、阈值电压分布及失效 bit 的物理位置。
- 必要时进行无损检查和物理失效分析。

## 案例二：MLCC 绝缘电阻下降

模拟现象：

- 回流焊后，部分 MLCC 出现绝缘电阻下降。
- 异常与板弯工序存在相关性。
- 外观检查不能稳定发现裂纹。

使用 `mlcc-insulation-failure-template.md` 完成调查。

建议验证方向：

- X-ray、声学显微镜、切片和染色渗透。
- 对照不同板弯应变、贴装方向和焊盘设计。
- 排查陶瓷裂纹、分层、端电极损伤和污染。
- 通过受控弯曲实验验证，而不是仅凭相关性结案。

## 完成标准

- 根因必须有对照实验支持。
- 临时遏制与永久纠正措施分开。
- 措施必须有责任人、期限和验证指标。
- 报告明确哪些是事实、推测和待验证事项。
